技术沉淀与复用 04

组件化开发

UI
通用 UI 组件库

CustomCard (标题/操作栏自定义), CustomTabs (响应式/横向滚动), DataCard (图标/增长率), StatCard (数据对比/图表集成/数字动画), Pad通用组件 (确认框/选择框/Toast)。通用组件在4个项目中复用，减少重复代码60%+。

SVG
设备图标组件体系

迁移至全局目录(src/components/DevicesIcons)，覆盖15+类设备（照明、空调、新风、门禁、电梯、停车、监控、消防、电表、环境监测、排风扇、一氧化碳、风阀、播放盒等），支持颜色主题切换和状态显示，实现跨页面、跨模块复用。

Biz
业务组件封装

门禁/车辆授权对话框、视频告警对话框、通用权限选择组件、播控系统组件（PlayboxIcon、programCard、FullScreenEditor）、能耗总览组件（月度用电统计、 用电联动分析、用电量对比统计、 用电排名等）、天气图标组件（132个彩色SVG图标）、全屏切换组件。

总结与展望

取得的成绩

  • 高效交付： 全年1100+次提交，覆盖4个项目，高频迭代。
  • 质量保障： 通过组件化开发和代码重构，减少重复代码60%+，可维护性提升。
  • 核心突破： 监控大屏智能轮播、Pad端楼控重构、设备卡片UI统一、组件化开发体系、综合视图系统、园区总览视图、地市运行报表系统、驾驶舱大屏系统、网格一张图大屏、智慧播控系统全新构建、能耗总览页面建设、图表轮播功能创新、天气图标组件封装。
  • 技术积累： 在视频播放、数据可视化、拖拽交互、组件化开发等方面积累丰富经验。

存在的不足

  • 需求理解与设计： 部分功能迭代频繁，技术方案设计阶段的前瞻性需要提升。
  • 架构设计能力： 面对复杂业务场景时，架构设计不够优雅，后期维护成本较高。
  • 自动化测试： 测试覆盖不足，需要逐步提升测试意识。
  • 文档编写： 缺少系统的技术文档，通用组件缺少完善的使用说明。
  • 时间管理： 有时过度追求技术完美而稍微影响整体进度。
